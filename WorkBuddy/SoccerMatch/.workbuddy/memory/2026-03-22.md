# 2026-03-22 工作日志

## 修复头像昵称闭环问题

### 问题分析
用户反馈头像昵称修改后在各页面展示不一致，经过检查发现以下问题：

1. **detail.js 的 saveUserInfo 方法**：使用 `openid` 作为 `_id` 保存，但没有保存 `openid` 字段，且没有同步更新本地存储
2. **profile.js 的 saveUserInfo 方法**：查询和保存逻辑不一致，可能导致重复记录
3. **app.js 的 getUserInfo**：只从本地缓存读取，没有从云端同步最新数据
4. **updateRegistration 云函数**：只更新 activities 集合，没有同步更新 users 集合

### 修复内容

#### 1. app.js - 启动时从云端同步用户信息
- 新增 `syncUserInfoFromCloud(openid)` 方法
- 登录成功后优先从云端 users 集合获取最新用户信息
- 同步更新 globalData 和本地存储

#### 2. detail.js - 完善用户信息保存
- 保存时添加 `_id` 和 `openid` 字段
- 使用 `positions` 数组替代 `position` 字符串
- 同时更新 `globalData` 和本地存储 `wx.setStorageSync`

#### 3. profile.js - 统一保存逻辑
- 使用 `doc(openid).set()` 方法统一创建/更新逻辑
- 确保 `_id` 和 `openid` 字段一致
- 数据格式与其他页面保持一致

#### 4. updateRegistration 云函数 - 同步用户信息
- 报名成功后自动同步用户信息到 users 集合
- 使用 `set` 方法确保记录存在或更新
- 不影响报名主流程（try-catch 包裹）

### 数据流闭环
```
用户修改头像昵称（我的页面/报名弹窗）
    ↓
保存到 users 集合（以 openid 为 _id）
    ↓
同时更新本地存储 globalData + wx.setStorageSync
    ↓
其他页面展示时从 users 集合查询最新数据
    ↓
下次启动时从云端同步到本地
```
