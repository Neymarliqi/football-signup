## 一、产品定位

**来踢球**是一款面向业余足球爱好者的微信小程序，用于足球活动的发布、报名、战术排布，以及**球队管理**。核心解决"约人踢球"这个场景，让组织者快速发活动、参与者和球队成员一键报名。

- **AppID**: `wx28d678b433d1626b`
- **技术栈**: 微信云开发（数据库 + 云函数）
- **云环境**: `cloud1-5gbn5i1p97239e9d`

## 二、用户角色

| 角色 | 说明 | 权限 |
|------|------|------|
| 未注册用户 | 未完成注册的用户 | 浏览活动列表、查看活动详情、查看战术板 |
| 注册用户 | 填写过头像+昵称的用户 | 浏览、报名、待定、请假、发布活动、查看战术板、编辑个人资料、设置位置偏好 |
| 球队成员 | 加入了某球队的用户 | 查看球队主页、参与球队活动 |
| 球队创建者 | 创建了某球队的用户 | 球队所有管理权限（编辑信息、管理成员、发布球队活动、处理申请） |
| 球队管理员 | 被创建者设为管理员的成员 | 管理成员、处理申请 |
| 活动创建者 | 发布了某活动的用户 | 编辑/取消该活动、查看详情、删除已结束/已取消的活动 |
| 管理员 | admins 集合中记录的用户 | 访问管理后台（活动/公告/球员管理） |

## 三、功能模块

### 3.1 首页（pages/index/index）

- **活动列表**: 展示所有活动，支持筛选（全部/未开始/进行中/已结束）
- **球队活动**: 展示用户所属球队的活动（球队Tab）
- **活动卡片信息**:
  - 状态标签（报名中/进行中/已结束/已取消）
  - 标题、日期、时间、地点、赛制、费用
  - 报名进度条（已报名/最大人数）
  - 前6名已报名球员头像+昵称+位置标签
  - 我的报名状态（已报名/待定/请假）
  - 球队标识（球队活动显示所属球队）
- **公告轮播**: 自动滚动显示最新5条活跃公告
- **操作**: 查看详情、战术板、编辑（创建者）、取消（创建者）、删除（创建者）
- **管理员入口**: 顶部⚙管理按钮（仅管理员可见）

### 3.2 活动详情（pages/activity/detail）

- **完整信息展示**: 标题、时间、地点（含腾讯地图导航）、赛制、人数、费用、场地类型、比赛说明
- **球队信息**: 如果是球队活动，显示所属球队Logo和名称
- **报名操作**:
  - 报名/待定/请假三选一
  - 未报名时需先确认阅读活动说明（3秒倒计时）
- **球员列表**: 按状态分组（已报名/待定/请假），显示头像、昵称、位置
- **战术板入口**: 跳转排兵布阵

### 3.3 创建/编辑活动（pages/activity/create）

- **发布活动**: 标题、日期、时间、地点（腾讯地图搜索+选点）、赛制、人数上限、费用、场地类型、比赛说明、允许待定开关
- **球队关联**:
  - 可选择发布到指定球队（球队活动）或公开（普通活动）
  - 球队创建者/管理员可发布球队活动
  - 球队活动分享后仅球队成员可见
- **编辑活动**: 创建者可在报名中状态下编辑活动信息

### 3.4 战术板（pages/tactics/tactics）

- **权限**: 所有人可查看，仅报名者+创建者可保存
- **球场可视化**: 绿色足球场 + 场地线条（中线、中圈、禁区、球门区）
- **球员管理**:
  - 替补区：显示所有未上场的已报名球员
  - 拖拽操作：替补→球场（上场）、场内拖动（移动位置）、场内→场外（下场）
- **保存**: 保存球员站位到数据库，实时同步

### 3.5 个人中心（pages/profile/profile）

- **个人信息**: 头像、昵称、位置偏好（首/备选位置）
- **统计数据**: 参与场次、进球、助攻（预留）
- **我的状态**: 位置偏好设置，支持从预设位置列表选择
- **参与记录**: 分页加载历史参与活动（服务端过滤，高性能）
- **编辑资料**: 修改头像（云存储方案）、昵称
- **我的球队**: 快速进入创建的球队和加入的球队列表
- **关于我们**: 版本号 v3.0.1

### 3.6 球队主页（pages/team/home）

- **封面+Logo**: 显示球队封面图和Logo
- **成员管理**: 查看成员列表，管理员可设置其他管理员/移除成员
- **活动列表**: 查看球队发布的活动
- **散客管理**: 查看曾报名但未入队的散客，可批量转入成员
- **申请管理**: 创建者/管理员可审批/拒绝加入申请
- **发布活动**: 创建者/管理员可发布球队活动
- **邀请成员**: 通过微信分享邀请加入
- **三态加载**: 进入页面时显示加载中，避免闪烁

### 3.7 球队创建（pages/team/create）

- **创建球队**: 封面图（可选）、Logo（必填）、球队名称（必填）、简介（可选）
- **加入方式**: 公开（直接加入）/ 审核（需审批）
- **自动成为创建者**: 创建者自动成为球队成员，角色为 creator

### 3.8 球队设置（pages/team/settings）

- **编辑信息**: 修改封面、Logo、名称、简介、加入方式
- **邀请成员**: 生成分享卡片
- **解散球队**: 仅创建者可解散

### 3.9 管理后台（pages/admin/admin）

- **活动管理**: 查看/编辑/删除活动
- **公告管理**: 发布/显示隐藏/删除公告
- **球员管理**: 查看所有注册球员（头像、昵称、位置、参与场次）

### 3.10 注册弹窗（全局组件 register-modal）

- **触发条件**: 仅在执行需要用户身份的操作时触发（报名/待定/请假/发布活动/编辑资料/位置偏好），不再自动拦截
- **核心原则**: "先浏览，后注册" — 所有页面均可正常浏览，只在需要身份时才引导注册
- **必填项**: 头像（chooseAvatar）+ 昵称（手动输入）+ 隐私协议勾选
- **关闭方式**: 右上角 ✕ 关闭按钮 + 点击遮罩层关闭（用户可拒绝注册继续浏览）
- **隐私保护**: 勾选框文字可跳转 /pages/privacy/privacy
- **注册后继续**: 注册成功后自动执行用户之前触发的操作（如报名→自动弹出活动描述确认）

## 四、数据概览

### 核心数据集合

| 集合 | 说明 | 关键字段 |
|------|------|----------|
| activities | 活动数据 | title, status, date, time, locationName, location, matchType, maxPlayers, fee, registrations[], createdBy, **teamId, teamName, visibility** |
| users | 用户信息 | nickName, **cloudPath**, avatarBase64, avatarUrl, positions[] |
| admins | 管理员 | openid |
| announcements | 公告 | content, active, createdBy |
| tactics | 战术板 | activityId, positions{openid:{x,y}} |
| **teams** | 球队主表 | name, logoPath, coverPath, description, joinMethod, creatorOpenid, memberCount |
| **team_members** | 球队成员 | teamId, openid, role(creator/admin/member), joinedAt |
| **team_applications** | 球队申请 | teamId, openid, status(pending/approved/rejected), appliedAt |

### 活动状态流转

```
open(报名中) → ongoing(进行中)
open(报名中) → finished(已结束)
open(报名中) → cancelled(已取消)
```

### 报名状态

```
confirmed(已报名) / pending(待定) / leave(请假)
```

### 球队成员角色

```
creator(创建者) / admin(管理员) / member(普通成员)
```

### 申请状态

```
pending(待审批) / approved(已通过) / rejected(已拒绝)
```

## 五、微信审核合规（v2.1.1）

### 先浏览后注册

- 所有页面（首页/详情/创建/个人中心）`onShow` 不再拦截注册
- 未注册用户可自由浏览活动列表、活动详情、战术板
- 注册弹窗有右上角 ✕ 关闭按钮，用户可拒绝注册

### 注册触发时机

| 操作 | 是否触发注册弹窗 |
|------|:---:|
| 浏览活动列表 | ❌ |
| 查看活动详情 | ❌ |
| 查看战术板 | ❌ |
| 浏览个人中心 | ❌ |
| 点击报名 | ✅ |
| 点击待定/请假 | ✅ |
| 点击发布活动（TabBar + 发布页提交） | ✅ |
| 点击编辑资料 | ✅ |
| 点击位置偏好 | ✅ |

### 三层缓存体系

```
内存缓存 → 本地 Storage → 数据库查询
```

| 缓存层 | 位置 | 生命周期 | 用途 |
|--------|------|----------|------|
| 内存缓存 | app.globalData.usersCache | 小程序存活期间（1分钟过期） | 秒级读取 |
| Storage 缓存 | wx.setStorageSync('user_${openid}') | 30天（cachedAt 超时清理） | 冷启动可用 |
| 数据库 | users 集合 | 永久 | 最终兜底 |

### 首页加载策略（stale-while-revalidate）

- 30秒内返回首页 → 用内存缓存秒渲染，后台静默更新用户信息
- 超过30秒 → 用用户缓存渲染活动列表，拉最新活动数据
- 冷启动 → 先读本地 Storage 缓存秒渲染，同时拉网络

### 永久登录状态

- 本地标记 `user_registered`（只有删除小程序才会消失）
- 注册判断不等网络，秒级响应，不会重复弹注册弹窗
