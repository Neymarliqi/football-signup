> 更新时间：2026-03-26

## 一、用户身份体系

```
首次打开小程序
      │
      ▼
wx.cloud.init() → 调用 getOpenid 云函数
      │
      ├── 成功 → openid 存入 globalData + localStorage
      └── 失败 → 读 localStorage 缓存（离线模式）
            │
            ▼
syncUserInfoFromCloud(openid)
      ├── users/{openid} 存在 → isGuest=false
      └── 不存在 → isGuest=true
            │
            ▼
checkAdmin(openid)
      ├── admins集合有记录 → isAdmin=true
      └── 无记录 → isAdmin=false
```

### 三种身份

| 身份 | 条件 | 能做什么 |
|------|------|----------|
| 游客 | 有openid，无users记录 | 浏览活动列表，**不能报名/发布/选位置** |
| 普通用户 | 有users记录（nickName+avatarUrl） | 所有功能 |
| 管理员 | admins集合中有记录 | 普通用户权限 + 管理后台入口 |

### 游客升级入口（3个）

1. **活动详情页** → 点击报名 → 弹出头像昵称弹窗 → 保存后自动继续报名
2. **创建活动页** → 点击发布 → 弹出头像昵称弹窗 → `app.createUser()` → 继续发布
3. **编辑资料页** → 主动填写头像昵称 → 保存后 `isGuest=false`

## 二、活动生命周期

```
              [创建者发布]
                    │
                    ▼
      ┌────── open（报名中）──────┐
      │          │                │
      │   用户可：报名/待定/请假    │
      │          │                │
   [创建者取消] [手动改状态]     [活动日期过期]
      │          │                │
      ▼          ▼                ▼
  cancelled   ongoing          effectiveStatus
  （已取消）  （进行中）         = finished（前端计算）
      │          │                │
      └──────────┼────────────────┘
                 ▼
           finished（已结束）
                 │
          [创建者可删除]
                 ▼
            （活动消失）
```

### 权限矩阵

| 操作 | open | ongoing | finished | cancelled |
|------|------|---------|----------|-----------|
| 报名/待定/请假 | ✅ 任何人 | ❌ | ❌ | ❌ |
| 取消报名 | ✅ 已报名者 | ❌ | ❌ | ❌ |
| 编辑活动 | ✅ 仅创建者 | ❌ | ❌ | ❌ |
| 取消活动 | ✅ 仅创建者 | ❌ | ❌ | ❌ |
| 删除活动 | ❌ | ❌ | ✅ 仅创建者 | ✅ 仅创建者 |

### effectiveStatus（关键逻辑）

活动数据库 status 不会自动流转。前端通过**日期对比**计算实际状态：

```javascript
// detail.js / create.js / admin.js 统一使用
if (status === 'finished' || activityDate < now) → effectiveStatus = 'finished'
else if (status === 'cancelled') → effectiveStatus = 'cancelled'
else → effectiveStatus = 'open'
```

## 三、各页面详细逻辑

### 1. 首页 (pages/index/index)

**功能**：活动列表 + 公告横幅 + 筛选器 + 创建/管理入口

**数据流**：
```
onShow → 判断距上次显示是否超过30秒
  ├── 是 → 全量 loadActivities() + loadAnnouncements()
  └── 否 → 仅 refreshUsersInfo()（刷新头像）
  → db.watch() 监听近7天活动实时变化
```

**核心逻辑**：
- **筛选器**：all / upcoming(open+未来日期) / ongoing / finished(cancelled+finished)
- **智能刷新**：30秒内回切不重新拉取，仅刷新用户头像
- **实时监听**：`db.watch()` 监听 createdAt > 7天前的活动，失败5秒后降级为全量拉取
- **取消活动**：前端直接 `db.update({status:'cancelled'})`，无需云函数
- **删除活动**：调用 `deleteActivity` 云函数（云端校验权限）
- **活动卡片**：显示前6人头像（限空间）、首选位置前2字、报名进度条

**跳转**：goDetail / goEdit / goCreate / goTactics / goAdmin

---

### 2. 活动详情 (pages/activity/detail)

**功能**：完整活动信息 + 报名/取消 + 球员列表 + 地图导航 + 分享

**数据流**：
```
onLoad(id) → db.doc(id).get() → fetchUsersWithCache(所有报名者)
         → processActivity() → db.watch() 实时监听
```

**报名流程（多重保护）**：
```
点击报名
  ├── 无用户信息 → 头像昵称弹窗 → 保存后继续
  └── 有用户信息 → 阅读确认弹窗（3秒倒计时）
        └── 倒计时结束 → 按钮激活 → 确认
              └── doRegister('confirmed') → updateRegistration 云函数

待定/请假：ActionSheet 选择 → 直接执行（无需阅读确认）
请假：ActionSheet → 请假原因输入框 → doRegister('leave', reason)
```

**权限**：
- 编辑：`isCreator && effectiveStatus === 'open' && activityDate > now`
- 取消：`isCreator && effectiveStatus === 'open'`
- 删除：`isCreator && (effectiveStatus === 'cancelled' || 'finished')`

**地图导航**：
```
有经纬度 → wx.openLocation()
无经纬度有地址 → wx.getLocation() 获取当前位置 → wx.openLocation()
无地址 → 提示未设置导航位置
```

**分享**：支持 `onShareAppMessage`（好友）和 `onShareTimeline`（朋友圈）

---

### 3. 创建/编辑活动 (pages/activity/create)

**功能**：活动表单 + 地图选点 + 赛制管理 + 截止时间

**两种模式**：
- **创建模式**：默认明天、19:00-21:00、默认免责声明、maxPlayers=16
- **编辑模式**：loadActivity(id) → 回填表单 → `updateActivity` 云函数保存

**游客拦截（双重检查）**：
```
submit() → 检查 globalData/localStorage 的 nickName
         → 无 → 头像昵称弹窗 → app.createUser() → submit(skipGuestCheck=true)
         → 有 → app.ensureUserCreated() 云端二次确认 → 继续
```

**地图选点**：腾讯地图选点插件 `plugin://chooseLocation/index`

**赛制类型**：预设6种（友谊赛/11人制/9人制/8人制/7人制/5人制）+ 每个活动独立自定义赛制列表

**截止时间**：5种预设（1天前/6小时前/1小时前/30分钟前/不限制）→ `activityDate - hours`

**表单校验**：title必填、date必填、startTime必填、locationName必填、matchType ≤ 10字

---

### 4. 战术板 (pages/tactics/tactics)

**功能**：可交互足球战术板，拖拽排兵布阵

**权限**：只有 `confirmed` 状态的报名者或创建者可访问，否则弹窗返回

**球员操作**：

| 操作 | 效果 |
|------|------|
| 点击替补球员 | 上场（球场底部） |
| 点击场上球员 | 选中（高亮） |
| 点击球场空白处 | 选中球员移动到该坐标 |
| 长按场上球员 | 下场（移回替补） |
| 拖拽球员到球场内 | 精确落点放置 |
| 拖拽球员出球场 | 下场 |

**坐标系**：相对球场容器的百分比（0-100），限制在 5%-95%

**保存**：`tactics` 集合，`{activityId, positions: {openid: {x, y}}}`

**onShow刷新**：强制刷新用户缓存保持头像最新，但保留当前已排好的坐标

---

### 5. 个人中心 (pages/profile/profile)

**功能**：用户信息展示/编辑 + 位置偏好 + 参与统计 + 最近5条历史

**位置偏好**：
- 多选最多3个，按选择顺序排序（order=1首, order=2备, order=3备）
- 数据格式：`[{value: 'ST', order: 1, label: '首'}, ...]`
- 保存后调用 `updateUserPosition` 云函数同步到所有已报名活动
- 游客不能选位置（引导至编辑资料页）

**头像更新**：3种方式（微信头像/相册/拍照），上传至云存储 `avatars/{openid}_{timestamp}.ext`

**统计数据（聚合查询）**：
```javascript
Promise.all([
  // 聚合查统计：match + filter + arrayElemAt + project
  // 查前5条：where('registrations.openid') + limit(5)
])
```

**智能加载**：先显示本地缓存（秒开）→ 后台静默更新 → 数据变化才setData

---

### 6. 参与记录 (pages/profile/history)

**功能**：分页展示全部参与历史 + 统计数据

**分页**：pageSize=20，skip/limit分页，hasMore判断

**统计**：聚合查询 `aggregate().match().project().end()`，服务端计算confirmed/pending/leave

**下拉刷新** + **上拉加载更多**

---

### 7. 编辑资料 (pages/profile/edit-profile/edit-profile)

**功能**：修改头像和昵称

**流程**：选择头像(微信open-type) + 输入昵称 → 验证 → 上传头像到云存储 → 先写本地 → 异步写云端(merge:true) → clearUserCache → 返回

**游客建立身份**：保存成功后 `isGuest=false`

---

### 8. 管理后台 (pages/admin/admin)

**功能**：活动列表 + 公告管理(增删改查) + 球员列表

**权限**：`onLoad` 检查 `isAdmin`，非管理员1.5秒后跳回

**公告管理**：创建(add) / 切换显示隐藏(update active) / 删除(remove)

**数据加载**：Promise.all 并行拉取 activities(30) + announcements(20) + users(50)

**待开发**：数据导出功能（占位提示）

---

### 9. 隐私保护 (pages/privacy/privacy)

纯静态页面，展示隐私保护指引文本。

## 四、云函数清单

| 函数 | 用途 | 权限校验 | 关键逻辑 |
|------|------|----------|----------|
| **getOpenid** | 获取openid | 无 | 返回 OPENID/APPID/UNIONID |
| **updateRegistration** | 报名/待定/请假 | 云端openid | 检查活动状态→满员检查→更新/追加registrations→同步users |
| **cancelRegistration** | 取消报名 | 云端openid | 检查活动状态→filter移除用户记录 |
| **updateActivity** | 编辑活动 | createdBy=openid | 状态检查→字段类型处理(Date/Number/Boolean) |
| **deleteActivity** | 删除活动 | ⚠️ 前端传openid | 检查createdBy→remove |
| **updateUserPosition** | 同步位置到活动 | 云端openid | 查所有参与活动→逐条更新registrations.position |
| **setAdmin** | 设置管理员 | 无 | add/remove admins集合记录 |
| **getMapSignature** | 地图API签名转发 | 无 | 参数排序→MD5签名→转发腾讯API |
| **tencentMapSearch** | 腾讯地图搜索 | 无 | 空目录，未实现 |

### ⚠️ 安全隐患

`deleteActivity` 接受前端传入的 `openid` 参数，而非从 `wxContext` 获取，理论上可伪造。建议改为 `updateActivity` 的方式使用 `wxContext.OPENID`。

## 七、关键设计模式

| 设计点 | 实现方式 |
|--------|----------|
| **实时数据** | 首页/详情页 `db.watch()` 监听，失败5秒后降级为全量拉取 |
| **用户缓存** | `app.globalData.usersCache`，1分钟过期，支持强制刷新 |
| **离线容灾** | openid/userInfo 写入 localStorage，云端失败读本地 |
| **本地优先** | 用户信息修改先写本地再异步同步云端（不阻塞UI） |
| **游客保护** | 报名/发布/选位置前检查 nickName，引导完善资料 |
| **权限双校验** | 前端判断 + 云函数服务端再次校验 |
| **位置兼容** | 全项目兼容旧格式（字符串）和新格式（`{value, order, label}`） |
| **满员防护** | 满员检查在云函数中进行，防止并发超员 |
| **effectiveStatus** | 前端基于日期实时计算，弥补DB状态未自动流转的问题 |
| **智能刷新** | 30秒内回切不重新拉取，仅刷新用户头像信息 |
