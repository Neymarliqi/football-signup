# 头像保存与查询优化总结

## 优化时间
2026-03-25

---

## 一、优化目标

1. **数据保存**：用户信息保存时，先保存到本地缓存，再异步保存到云端
2. **数据查询**：首页、报名详情页、战术板统一使用全局缓存系统查询用户头像

---

## 二、优化内容

### 1. 用户信息保存逻辑优化

#### 1.1 个人中心编辑页面（`pages/profile/edit-profile/edit-profile.js`）

**优化前：**
- 上传头像后直接保存到云端
- 云端保存失败时用户体验差

**优化后：**
```javascript
// 1. 先获取 openid
const openid = app.globalData.openid || wx.getStorageSync('openid')

// 2. 如果上传了新头像，先上传到云存储
if (tempAvatarUrl) {
  const uploadRes = await wx.cloud.uploadFile({...})
  finalAvatarUrl = uploadRes.fileID
}

// 3. 构建更新后的用户信息
const updatedUserInfo = {...}

// 4. 更新本地缓存（优先）
app.globalData.userInfo = updatedUserInfo
wx.setStorageSync('userInfo', updatedUserInfo)

// 5. 异步保存到云端（不影响用户体验）
db.collection('users').doc(openid).update({...}).catch(err => {
  // 云端保存失败不影响用户体验
})
```

**优点：**
- ✅ 保存响应更快（无需等待云端）
- ✅ 即使云端失败，本地数据也不会丢失
- ✅ 下次打开页面使用本地最新数据

---

#### 1.2 首次报名弹窗（`pages/activity/detail.js` - `saveUserInfo()`）

**优化前：**
- 先同步保存到云端，再更新本地缓存
- 云端保存失败时整个流程中断

**优化后：**
```javascript
// 1. 先获取 openid
let openid = app.globalData.openid || wx.getStorageSync('openid')

// 2. 如果选择了头像，上传到云存储
if (tempAvatarUrl && !tempAvatarUrl.startsWith('cloud://')) {
  const uploadRes = await wx.cloud.uploadFile({...})
  finalAvatarUrl = uploadRes.fileID
}

// 3. 构建用户信息
const userInfo = {...}

// 4. 更新本地缓存（优先）
app.globalData.userInfo = userInfo
wx.setStorageSync('userInfo', userInfo)

// 5. 异步保存到云端（不影响用户体验）
db.collection('users').doc(openid).update({...}).catch(err => {
  // 如果文档不存在，使用 set 创建
  if (err.errCode === -502001 || err.errCode === -502005) {
    db.collection('users').doc(openid).set({...})
  }
})
```

**优点：**
- ✅ 报名流程不会被云端保存失败影响
- ✅ 用户头像和昵称立即生效
- ✅ 保证了用户体验的流畅性

const batchSize = 20
for (let i = 0; i < userIds.length; i += batchSize) {

## 三、全局缓存系统（`app.js`）

### 缓存机制

```javascript
// app.js - fetchUsersWithCache()
async fetchUsersWithCache(userIds) {
  const { usersCache, cacheExpireTime } = this.globalData
  const now = Date.now()
  const result = {}
  const uncachedIds = []

  // 1. 先从缓存中查找
  userIds.forEach(id => {
    const cached = usersCache[id]
    if (cached && (now - cached.timestamp < cacheExpireTime)) {
      // 缓存有效，使用缓存
      result[id] = cached.data
    } else {
      // 缓存过期或不存在，需要查询
      uncachedIds.push(id)
    }
  })

  // 2. 批量查询未缓存的用户
  if (uncachedIds.length > 0) {
    const batchSize = 20
    for (let i = 0; i < uncachedIds.length; i += batchSize) {
      const batch = uncachedIds.slice(i, i + batchSize)
      const res = await db.collection('users').where({
        openid: db.command.in(batch)
      }).get()

      // 3. 更新缓存
      res.data.forEach(user => {
        result[user.openid] = user
        usersCache[user.openid] = {
          data: user,
          timestamp: now
        }
      })
    }
  }

  return result
}
```

### 缓存配置

- **缓存过期时间：** 5 分钟（`cacheExpireTime = 5 * 60 * 1000`）
- **批量查询大小：** 每批 20 个用户
- **缓存键：** 用户 `openid`

### 缓存清除

当用户信息更新时，清除对应缓存：

```javascript
// 清除单个用户的缓存
clearUserCache(userId) {
  if (userId && this.globalData.usersCache) {
    delete this.globalData.usersCache[userId]
  }
}
```

## 五、注意事项

### 1. 数据库查询规范

**必须使用 `openid` 字段查询：**
```javascript
// ✅ 正确：使用 openid
const res = await db.collection('users').where({
  openid: db.command.in(batch)
}).get()

// ❌ 错误：使用 _id（会导致缓存查询失败）
const res = await db.collection('users').where({
  _id: db.command.in(batch)
}).get()
```

### 2. 缓存键规范

**缓存 key 必须使用 `openid`：**
```javascript
// ✅ 正确：使用 openid
result[user.openid] = user
usersCache[user.openid] = { data: user, timestamp: now }

// ❌ 错误：使用 _id
result[user._id] = user
usersCache[user._id] = { data: user, timestamp: now }
```

### 3. 本地存储优先

**保存时先存本地，再异步存云端：**
```javascript
// 1. 先保存到本地
app.globalData.userInfo = userInfo
wx.setStorageSync('userInfo', userInfo)

// 2. 异步保存到云端（失败不影响用户体验）
db.collection('users').doc(openid).update({...}).catch(...)
```
