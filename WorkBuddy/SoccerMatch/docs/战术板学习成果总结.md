## 一、功能概述

足球战术板是"约球助手"小程序的核心功能之一，用于**可视化展示球员在场上的位置安排**，支持拖拽调整，帮助球队组织者进行战术布置。

### 核心功能点
- ✅ 可视化足球场展示
- ✅ 球员Token拖拽上场
- ✅ 场上位置自由调整
- ✅ 替补席管理
- ✅ 战术保存与加载
- ✅ 权限控制（仅已报名用户可见）

## 三、核心实现原理

### 3.1 坐标系统

#### 百分比坐标 vs 像素坐标
| 类型 | 用途 | 范围 | 说明 |
|------|------|------|------|
| 百分比坐标 | 数据存储 | 0-100 | 适配不同屏幕尺寸 |
| 像素坐标 | 渲染/拖拽 | 实际像素 | movable-view使用 |

#### 坐标转换公式
```javascript
// 百分比 → 像素（显示时用）
xPx = (x / 100) * fieldWidth - 50  // 50是token宽度的一半
yPx = (y / 100) * fieldHeight - 50

// 像素 → 百分比（保存时用）
x = ((xPx + 50) / fieldWidth) * 100
y = ((yPx + 50) / fieldHeight) * 100
```

### 3.2 拖拽实现

使用微信小程序的 `movable-area` + `movable-view` 组件实现拖拽：

```xml
<!-- 场地容器 -->
<movable-area class="field" id="field">
  <!-- 可拖拽的球员Token -->
  <movable-view wx:for="{{onFieldPlayers}}"
                x="{{item.xPx}}" 
                y="{{item.yPx}}"
                direction="all"
                bindtouchend="onPlayerDragEnd">
    <image src="{{item.avatarUrl}}" />
    <text>{{item.shortName}}</text>
  </movable-view>
</movable-area>
```

### 3.3 拖拽逻辑流程

```
替补席拖拽上场：
1. 监听 bindtouchend 事件
2. 获取拖拽后的屏幕坐标
3. 计算相对于球场的坐标
4. 判断是否在球场范围内
5. 在范围内 → 添加到 onFieldPlayers，从 benchPlayers 移除
6. 不在范围内 → 回到原位

场上拖拽调整：
1. 监听 bindtouchend 事件
2. 获取最终位置 xPx, yPx
3. 检查是否拖出场地
4. 拖出 → 移回替补席
5. 未拖出 → 更新坐标，转换为百分比保存
```

## 六、关键代码片段

### 6.1 加载战术数据
```javascript
async loadTactics() {
  // 1. 获取活动数据
  const res = await db.collection('activities').doc(activityId).get()
  
  // 2. 检查权限
  const isCreator = act.createdBy === openid
  const isParticipant = registrations.some(r => r.openid === openid && r.status === 'confirmed')
  this.setData({ canEdit: isCreator || isParticipant })
  
  // 3. 加载已保存的战术
  const tacticRes = await db.collection('tactics').where({ activityId }).get()
  const savedPositions = tacticRes.data[0]?.positions || {}
  
  // 4. 构建球员列表
  const allPlayers = confirmedRegistrations.map(r => ({
    ...r,
    xPx: savedPositions[r.openid] ? (savedPositions[r.openid].x / 100) * fieldWidth - 50 : null,
    yPx: savedPositions[r.openid] ? (savedPositions[r.openid].y / 100) * fieldHeight - 50 : null,
    isOnField: !!savedPositions[r.openid]
  }))
}
```

### 6.2 保存战术
```javascript
async saveTactics() {
  const positions = {}
  onFieldPlayers.forEach(p => {
    positions[p.openid] = { x: p.x, y: p.y }
  })
  
  const existing = await db.collection('tactics').where({ activityId }).get()
  
  if (existing.data.length > 0) {
    await db.collection('tactics').doc(existing.data[0]._id).update({
      data: { positions, updatedAt: db.serverDate() }
    })
  } else {
    await db.collection('tactics').add({
      data: { activityId, positions, createdAt: db.serverDate() }
    })
  }
}
```

### 6.3 跨组件坐标计算
```javascript

query.select('#benchArea').boundingClientRect()

  // 计算球员相对于球场的坐标
```

## 八、经验总结

### 8.1 技术要点
1. **坐标系统**：使用百分比存储，像素渲染，确保多设备适配
2. **拖拽实现**：movable-view 组件比 touch 事件更高效
3. **跨组件定位**：使用 `boundingClientRect` 计算相对位置
4. **数据同步**：实时更新数据，拖拽结束时统一保存

### 8.2 设计要点
1. **深色主题**：符合运动类应用风格，减少视觉疲劳
2. **清晰层次**：球场 → 球员Token → 替补席，层次分明
3. **即时反馈**：拖拽有视觉反馈，保存有成功提示
4. **权限明确**：未报名用户无法查看，避免信息泄露

### 8.3 可优化方向
1. **阵型预设**：支持4-3-3、4-4-2等经典阵型一键布置
2. **位置标签**：显示GK、CB、CM等位置缩写
3. **动画效果**：拖拽时有更流畅的动画
4. **多战术保存**：支持保存多套战术方案
