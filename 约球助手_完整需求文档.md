## 一、产品概述

### 1.1 产品定位
约球助手是一款基于微信小程序的足球活动报名管理工具，帮助球队组织者发布活动、管理报名、安排战术，帮助球员快速报名、查看活动详情。

### 1.2 核心功能模块
| 模块 | 功能描述 |
|------|----------|
| 活动管理 | 创建、编辑、取消、删除活动 |
| 报名系统 | 报名/待定/请假三种状态，支持取消报名 |
| 人员管理 | 查看已报名/待定/请假人员列表 |
| 战术板 | 可视化阵型安排，支持拖拽调整位置 |
| 公告系统 | 管理员发布活动公告 |
| 用户系统 | 头像昵称管理、位置信息 |

## 二、用户角色与权限

### 2.1 用户身份定义

| 身份 | 判断条件 | 说明 |
|------|---------|------|
| **发布者 (Creator)** | `activity.createdBy === openid` | 活动的创建者，拥有管理权限 |
| **参与者 (Participant)** | `registrations.find(r => r.openid === openid)` | 已报名/待定/请假的人 |
| **普通用户** | 以上都不是 | 只是浏览者，可以报名 |
| **管理员** | `app.globalData.isAdmin === true` | 系统管理员，拥有全部权限 |

### 2.2 权限矩阵

#### 发布者权限
| 操作 | 条件 | 说明 |
|------|------|------|
| 编辑活动 | `isCreator && status === 'open'` | 只有报名中可编辑 |
| 取消活动 | `isCreator && status === 'open'` | 只有报名中可取消 |
| 删除活动 | `isCreator && (status === 'cancelled' \|\| status === 'finished')` | 取消或结束后可删除 |

#### 参与者权限
| 操作 | 条件 | 说明 |
|------|------|------|
| 报名 | `!isCreator && !myStatus && status === 'open'` | 非发布者、未报名、活动进行中 |
| 取消报名 | `myStatus && status === 'open'` | 已报名且活动进行中 |
| 修改状态 | `myStatus && status === 'open'` | 已报名可在confirmed/pending/leave间切换 |
| 查看战术板 | `myStatus === 'confirmed'` | 已报名人员可见 |

## 三、活动状态管理

### 3.1 活动状态定义

| 状态 | 值 | 显示文案 | 样式类 | 说明 |
|------|-----|---------|--------|------|
| 报名中 | `open` | 报名中 | `tag-green` | 可报名、可编辑 |
| 进行中 | `ongoing` | 进行中 | `tag-blue` | 活动已开始 |
| 已结束 | `finished` | 已结束 | `tag-gray` | 活动时间已过 |
| 已取消 | `cancelled` | 已取消 | `tag-red` | 发布者取消 |

### 3.2 状态流转图

```
创建活动 → open（报名中）
  ↓
到达活动时间 → ongoing（进行中）
  ↓
活动结束 → finished（已结束）

open（报名中）→ cancelled（已取消）[发布者可取消]
```

### 3.3 状态判断逻辑

```javascript
const now = new Date()
const actDate = new Date(activity.activityDate)

if (activity.status === 'finished' || actDate < now) {
  statusText = '已结束'
  statusClass = 'tag-gray'
  status = 'finished'
} else if (activity.status === 'ongoing') {
  statusText = '进行中'
  statusClass = 'tag-blue'
  status = 'ongoing'
} else if (activity.status === 'cancelled') {
  statusText = '已取消'
  statusClass = 'tag-red'
  status = 'cancelled'
} else {
  statusText = '报名中'
  statusClass = 'tag-green'
  status = 'open'
}
```

## 四、报名状态管理

### 4.1 报名状态定义

| 状态 | 值 | 显示文案 | 样式类 | 说明 |
|------|-----|---------|--------|------|
| 已报名 | `confirmed` | ✅ 已报名 | `tag-green` | 确认参加 |
| 待定 | `pending` | ⏳ 待定 | `tag-yellow` | 暂时不确定 |
| 请假 | `leave` | 🙅 请假 | `tag-red` | 无法参加 |
| 未报名 | `null` | - | - | 未参与 |

### 4.2 报名状态流转

```
未报名
  ├──→ confirmed（报名）
  ├──→ pending（待定）
  └──→ leave（请假）

confirmed
  ├──→ pending（改为待定）
  ├──→ leave（请假）
  └──→ 取消报名

pending
  ├──→ confirmed（确认报名）[需检查满员]
  ├──→ leave（请假）
  └──→ 取消报名

leave
  ├──→ confirmed（改为报名）[需检查满员]
  ├──→ pending（改为待定）
  └──→ 取消报名
```

### 4.3 满员检查逻辑

- 只有 `confirmed` 状态需要检查是否满员
- 从 `pending` 或 `leave` 改为 `confirmed` 时，需检查当前 confirmed 人数是否 >= maxPlayers
- 满员时显示提示："⚠️ 当前人数已满！可选择待定，候补排队"

#### 5.6.3 用户信息同步机制

**数据流**：
```
用户修改信息（profile页）
    ↓
保存到 users 集合（openid 作为 _id）
    ↓
更新 globalData + 本地存储
    ↓
各页面从 users 集合查询最新数据
    ↓
下次启动时从云端同步到本地
```

**关键代码**：
```javascript
// 保存用户信息（profile.js）
await db.collection('users').doc(openid).set({
  data: {
    openid: openid,
    nickName: userInfo.nickName,
    avatarUrl: userInfo.avatarUrl,
    positions: userInfo.positions || [],
    updatedAt: db.serverDate()
  }
})

// 启动时同步（app.js）
await db.collection('users').doc(openid).get()
// 使用云端数据覆盖本地缓存
```

## 六、云函数接口

### 6.1 updateRegistration - 更新报名状态

**功能**：报名/修改状态/请假，同时同步用户信息到 users 集合

**参数**：
```javascript
{
  activityId: string,    // 活动ID
  status: string,        // 'confirmed' | 'pending' | 'leave'
  leaveReason: string,   // 请假原因（可选）
  nickName: string,      // 昵称
  avatarUrl: string,     // 头像URL
  position: array        // 位置偏好数组（可选）
}
```

**逻辑**：
1. 获取活动数据
2. 检查是否已报名
3. 满员检查（仅confirmed状态）
4. 更新或新增报名记录到 activities
5. **同步更新 users 集合**（确保头像昵称闭环）
6. 返回操作结果

**用户信息同步逻辑**：
```javascript
// 报名成功后自动同步到 users 集合
await db.collection('users').doc(openid).set({
  data: {
    openid: openid,
    nickName: nickName,
    avatarUrl: avatarUrl,
    positions: position || [],
    updatedAt: db.serverDate()
  }
})
```

**返回**：
```javascript
{ success: true, message: '操作成功' }
{ success: false, error: 'FULL', message: '报名人数已满' }
```

### 6.2 cancelRegistration - 取消报名

**功能**：完全取消报名（从列表中移除）

**参数**：
```javascript
{
  activityId: string
}
```

**逻辑**：
1. 获取活动数据
2. 过滤掉当前用户的报名记录
3. 更新数据库

| open | 未报名 | 报名(confirmed/pending/leave)、查看、分享 |
| open | confirmed | 改为pending、改为leave、取消报名、查看战术板、分享 |
| open | pending | 改为confirmed、改为leave、取消报名、查看、分享 |
| open | leave | 改为confirmed、改为pending、取消报名、查看、分享 |
| ongoing | 任意 | 查看、分享（不可修改） |
| finished | 发布者 | 删除、查看 |
| finished | 参与者 | 查看 |
| cancelled | 发布者 | 删除、查看 |
| cancelled | 参与者 | 查看 |

### 9.2 按钮显示状态穷举

**首页卡片按钮：**

| 条件 | 显示按钮 |
|------|----------|
| 所有活动 | 查看详情 |
| myStatus === 'confirmed' | 查看详情 + 战术板 |

**详情页底部按钮：**

| myStatus | 显示按钮 |
|----------|----------|
| null | 报名参加 + 暂时待定 + 我要请假 |
| confirmed | 改为待定 + 请假 + 取消报名 |
| pending | 确认报名 + 请假 + 取消 |
| leave | 改为报名 + 改为待定 + 取消 |

### 9.3 满员状态影响

| 场景 | 行为 |
|------|------|
| 满员时点击"报名参加" | 按钮禁用，显示满员提示 |
| 满员时点击"确认报名"（从pending） | 按钮禁用 |
| 满员时点击"改为报名"（从leave） | 按钮禁用 |
| 满员时显示 | "⚠️ 当前人数已满！可选择待定，候补排队" |
