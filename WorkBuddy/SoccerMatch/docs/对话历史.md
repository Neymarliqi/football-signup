## 二、详情页功能优化

### 2.1 地图导航优化
**需求**：点击地址直接调起地图导航

**实现方案**：
修改 `pages/activity/detail.js`：
```javascript
openMap() {
  const { activity } = this.data
  if (activity.latitude && activity.longitude) {
    wx.openLocation({
      latitude: activity.latitude,
      longitude: activity.longitude,
      scale: 18,
      name: activity.locationName || '踢球地点',
      address: activity.location || ''
    })
  }
}
```

**技术要点**：
- 使用 `wx.openLocation` 调起微信内置地图（基于腾讯地图）
- 不需要额外申请腾讯地图 API Key
- 坐标系要求：gcj02（`wx.chooseLocation` 返回的经纬度已符合）

### 2.3 首次报名头像昵称弹窗
**需求**：用户首次报名时引导填写头像和昵称

**实现方案**：
修改 `pages/activity/detail.js`：
```javascript
data: {
  showUserInfoModal: false,
  tempAvatarUrl: '',
  tempNickName: '',
  defaultAvatarUrl: 'https://mmbiz.qpic.cn/mmbiz/icTdbqWNOwNRna42FI242Lcia07jQodd2FJGIYQfG0LAJGFxM4FbnQP6yfMxBgJ0F3nRn9FQGjU4YPmt5ogAApw/640'
},

onChooseAvatar(e) {
  this.setData({ tempAvatarUrl: e.detail.avatarUrl })
},

onNickNameInput(e) {
  this.setData({ tempNickName: e.detail.value })
},

async saveUserInfo() {
  const { tempAvatarUrl, tempNickName } = this.data
  
  // 上传头像到云存储
  const uploadRes = await wx.cloud.uploadFile({
    cloudPath: `avatars/${app.globalData.openid}.jpg`,
    filePath: tempAvatarUrl
  })
  
  // 保存到数据库
  await db.collection('users').doc(app.globalData.openid).set({
    data: {
      nickName: tempNickName,
      avatarUrl: uploadRes.fileID,
      updatedAt: new Date()
    }
  })
  
  // 更新全局状态
  app.globalData.userInfo = {
    nickName: tempNickName,
    avatarUrl: uploadRes.fileID
  }
  
  this.setData({ showUserInfoModal: false })
}
```

**技术要点**：
- 使用微信最新头像组件：`<button open-type="chooseAvatar">`
- 使用微信最新昵称组件：`<input type="nickname">`
- 头像上传到云存储，fileID 保存到数据库
- 弹窗采用底部抽屉式动画（`slideUp`）

**遇到的问题**：

**问题1：Git 锁文件导致卡住**
```
fatal: Unable to create '/Users/Neymar/.git/index.lock': File exists.
```

**原因**：有一个 `git add -A` 进程僵死 17 分钟，创建了锁文件

**解决方案**：
```bash
# 杀掉僵死进程
kill -9 99493

# 删除锁文件
rm ~/.git/index.lock
```

---

**问题2：GitHub Token 认证**
```
remote: Password authentication is not supported for Git operations.
```

**原因**：GitHub 在 2021 年已禁用密码认证

**解决方案**：
- 需要生成 Personal Access Token（PAT）
- 在 https://github.com/settings/tokens 生成 Token
- 推送时用 Token 代替密码

---

**问题3：仓库结构混乱**
**原因**：初始时 Git 仓库在 `WorkBuddy` 目录，追踪了 `20260320162755` 和 `SoccerMatch` 两个文件夹

**解决方案**：
```bash
# 从 Git 追踪中删除旧文件夹
git rm -r --cached 20260320162755

# 确保 SoccerMatch 被追踪
git add SoccerMatch/

# 提交并推送
git commit -m "删除旧文件夹，保留SoccerMatch"
git push origin master
```

## 七、常见问题排查

### 云开发相关问题

**问题：云函数调用失败**
- 检查云函数是否上传成功
- 查看云函数日志
- 确认环境 ID 配置正确

**问题：数据库权限错误**
- 检查数据库权限设置
- 确认用户已登录（openid 已获取）

**问题：云存储上传失败**
- 检查云存储权限
- 确认文件大小在限制内（1MB）

### 小程序相关问题

**问题：地图无法打开**
- 检查是否申请了位置权限
- 确认经纬度格式正确
- 在真机上测试（模拟器可能不支持地图）

**问题：头像选择失败**
- 检查是否使用了正确的组件 `open-type="chooseAvatar"`
- 在真机上测试（模拟器不支持）
