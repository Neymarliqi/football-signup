## 四、微信小程序拖拽技术最佳实践

### 4.1 movable-view 组件详解

**基本结构**：
```xml
<movable-area class="area">
  <movable-view class="view" direction="all">可拖拽元素</movable-view>
</movable-area>
```

**核心属性**：
| 属性名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `direction` | string | - | all/vertical/horizontal/none |
| `inertia` | boolean | false | 惯性滑动效果 |
| `out-of-bounds` | boolean | false | 允许拖出边界 |
| `x` | number/string | - | 初始x轴位置(px) |
| `y` | number/string | - | 初始y轴位置(px) |
| `damping` | number | 20 | 阻尼系数 |
| `friction` | number | 2 | 摩擦系数 |
| `disabled` | boolean | false | 是否禁用 |
| `scale` | boolean | false | 双指缩放 |
| `animation` | boolean | true | 是否使用动画 |

**事件处理**：
- `bindchange`：拖动/缩放过程中触发
- `bindscale`：缩放过程中触发
- `htouchmove` / `vtouchmove`：横向/纵向触摸移动

### 4.2 两种拖拽实现方式对比

#### 方式一：movable-view 组件
**适用场景**：简单、快速实现拖拽，独立悬浮元素

**优点**：
- 官方组件，API简单
- 支持direction="all"全方向拖拽
- 拖拽流畅，体验好

**缺点**：
- 会遮挡同层级其他元素
- 无法灵活控制层级和交互

#### 方式二：绝对定位 + touch事件
**适用场景**：复杂布局、需与滚动内容协同

**优点**：
- 布局灵活，可嵌入任何位置
- 支持滚动页面内拖拽
- 适配性强

**缺点**：
- 需手动处理触摸事件
- 实现复杂度高
- 容易因布局计算导致卡顿

### 4.3 最佳实践建议

| 维度 | 推荐做法 |
|------|----------|
| **场景选择** | 独立悬浮按钮用movable-view；滚动页面复杂布局用绝对定位 |
| **性能优化** | 控制movable-area范围；绝对定位优化坐标计算，避免频繁setData |
| **兼容性** | movable-view官方组件兼容性有保障；绝对定位需自行处理机型适配 |
| **交互扩展** | 拖拽排序、边界检测、吸附效果建议基于绝对定位自行封装 |

### 4.4 注意事项
1. **容器必须设置宽高**：movable-area必须设置width和height
2. **movable-view必须设置宽高**：样式表中必须定义
3. **定位方式**：默认使用绝对定位，位置由x、y属性控制
4. **手势冲突**：使用catch事件阻止冒泡
5. **边界处理**：允许拖出边界时设置回弹动画

## 五、足球战术板交互设计最佳实践总结

### 5.1 核心交互模式

#### 1. 阵型快速布阵
- 提供预设阵型库（4-4-2、4-3-3、3-5-2等）
- 一键加载初始球员位置
- 支持11人制、8人制、5人制

#### 2. 球员拖拽操作
- **替补→场上**：拖拽替补球员进入球场区域即上场
- **场上移动**：自由拖拽调整位置
- **场上→替补**：拖拽出球场边界即下场
- **位置自动排序**：替补席球员始终保持有序排列

#### 3. 战术路线绘制（高级功能）
- 虚线箭头表示跑位路线
- 实线箭头表示传球路线
- 支持曲线、直线多种样式
- 可清除、重置路线

#### 4. 视图操作
- 缩放：双指缩放查看细节
- 旋转：三维视角旋转（高级）
- 截图分享：生成战术图分享

### 5.4 关键交互细节

#### 1. 拖拽检测逻辑
```javascript
// 使用boundingClientRect获取实际位置
const query = this.createSelectorQuery()
query.select('#field').boundingClientRect()
query.select('#benchArea').boundingClientRect()
query.exec((res) => {
  const fieldRect = res[0]
  const benchRect = res[1]
  
  // 计算球员屏幕坐标
  const playerScreenX = benchRect.left + x + offsetX
  const playerScreenY = benchRect.top + y + offsetY
  
  // 检查是否在球场内
  const inField = playerScreenX >= fieldRect.left && 
                  playerScreenX <= fieldRect.right &&
                  playerScreenY >= fieldRect.top && 
                  playerScreenY <= fieldRect.bottom
})
```

#### 2. 自动排序逻辑
```javascript
// 替补球员始终保持有序
const sortedBenchPlayers = benchPlayers.map((p, i) => ({
  ...p,
  benchX: i * 130  // 固定间距
}))
```

#### 3. 边界处理
- 球员拖拽出球场边界自动回到替补席
- 替补球员未拖进场自动回到原位
- 位置计算使用百分比，适配不同屏幕

### 5.5 性能优化建议

1. **避免频繁setData**：批量更新数据
2. **控制movable-area范围**：不要设置过大
3. **图片优化**：球员头像使用合适尺寸
4. **懒加载**：大量球员时考虑分页
5. **防抖处理**：拖拽结束后再保存位置

### 5.6 用户体验细节

1. **视觉反馈**：
   - 拖拽时球员放大或高亮
   - 进入可放置区域时视觉提示
   - 保存成功toast提示

2. **操作引导**：
   - 首次使用显示引导提示
   - 空状态显示操作说明
   - 权限不足时显示"仅查看"提示

3. **容错处理**：
   - 网络失败重试机制
   - 数据加载失败提示
   - 操作撤销功能（可选）
